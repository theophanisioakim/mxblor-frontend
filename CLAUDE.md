## Claude Code specifics

The shared rules in `AGENTS.md` are authoritative. The items below only cover Claude-Code-only
features in this repo.

### Slash commands (`.claude/commands/`)

- `/build` — `pnpm build` (turbo build, all workspaces).
- `/typecheck` — `pnpm typecheck`; pass a workspace to narrow (e.g. `/typecheck @workspace/ui`).
- `/lint` — `pnpm lint`.
- `/format` — `pnpm format` then report what changed.
- `/generate` — `pnpm generate` (re-run Orval to refresh `api-client/src/generated`).
- `/check` — `pnpm check:all` (generate → format → typecheck → lint → build — the full gate).
- `/add-ui` — scaffold a new cross-platform primitive in `@workspace/ui` (web `.tsx` + native
  `.native.tsx` + `index.ts`) wrapping the matching `web-ui`/`native-ui` component.

### Subagents (`.claude/agents/`)

- `architecture-reviewer` — audits a diff for boundary violations: `web-ui`/`native-ui` imported
  outside `ui`, cross-package relative imports, edits to vendored/generated code, version literals
  that should be in the catalog. Use before finishing any multi-file change.
- `ui-sync-reviewer` — checks that every `@workspace/ui` primitive has a synchronized web + native
  variant with matching exports/props/behavior, and is registered in `index.ts`.

### Skills (`.claude/skills/`) — invoke with `/name`, or Claude auto-invokes by description

- `/add-ui-component <Name>` — adds a synchronized cross-platform component to `@workspace/ui`
  wrapping the `web-ui` (shadcn) and `native-ui` (rnr) implementations, with a parity checklist.
- The `.claude/skills/` entries are **thin stubs** — the canonical, tool-agnostic skill bodies live
  in `.agents/skills/` (Agent Skills standard, also read by Codex/Copilot/Gemini/Cursor). Edit the
  `.agents/skills/` copy, never the stub. `.agents/skills/add-screen/` documents the end-to-end
  "new screen through every layer" procedure.

### Rules (`.claude/rules/`) — auto-loaded by file path, no invocation needed

Path-gated guidance that loads when you open a matching file:

- `vendored-ui-components.md` → `packages/web-ui/**`, `packages/native-ui/**`
- `ui-abstraction.md` → `packages/ui/src/components/**`
- `generated-api.md` → `packages/api-client/src/generated/**`
- `cross-platform.md` → `**/*.native.{ts,tsx}`
- `page-layout.md` → `packages/app/src/screens/**`, `apps/web/app/**`, `apps/native/src/app/**`
- `typescript-style.md` → `**/*.{ts,tsx}`
  These are guidance (like CLAUDE.md), not enforcement — the hook + permissions in `settings.json` are
  the enforced layer.

### Hooks (configured in `.claude/settings.json`)

- **PreToolUse guard** (`.agents/hooks/guard-protected-paths.mjs` — Node, cross-platform, shared
  with Codex via `.codex/hooks.json`) denies edits under
  `**/node_modules/**`, `**/.next/**`, `**/.turbo/**`, `**/dist/**`, `**/build/**`, and
  `packages/api-client/src/generated/**` (Orval-owned); and **asks before editing** vendored
  component files under `packages/web-ui/src/components/**` and `packages/native-ui/src/components/**`
  (regenerated by `pnpm shadcn-update` / `pnpm rnr-update`). See `AGENTS.md` §4–§5.
- **Execution-policy denies** (`permissions.deny` in `.claude/settings.json`) enforce `AGENTS.md`
  §0: the process-starting commands — `pnpm dev` / `run dev` / `turbo dev`, `pnpm --filter web dev`,
  `pnpm --filter native start`, the native `prebuild`/`android`/`ios` builds, and every `test:e2e`
  variant (Bash + PowerShell) — are **denied**. The agent may build, typecheck, lint, format, run
  `pnpm generate`, and run the **unit** `test` suite; hand any dev-server / device / E2E run to the
  user.

### Auto-loaded per-package rules

Each package/app has an `AGENTS.md` (canonical rules) and a `CLAUDE.md` stub that imports it. When
you read/edit a file in, say, `packages/ui/`, Claude Code pulls in `packages/ui/CLAUDE.md` →
`packages/ui/AGENTS.md`. **Edit the `AGENTS.md`, not the stub.**

### Conventions reminder

- Local dev is Windows **and** Linux: always run scripts via `pnpm`; in PowerShell use `$env:VAR`,
  `$null`, backtick line-continuation.
- Versions live in the `catalog:` (`pnpm-workspace.yaml`), not in individual `package.json` files.
- Don't `git commit`/`push` unless asked; never bypass the husky hooks with `--no-verify`. Git hooks
  (husky: pre-commit lint-staged, pre-push full gate) and CI are described in `AGENTS.md` §11 — these
  are separate from the Claude `PreToolUse` guard above.
