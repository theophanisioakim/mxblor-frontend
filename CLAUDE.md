## Claude Code specifics

The shared rules in `AGENTS.md` are authoritative. The items below only cover Claude-Code-only
features in this repo.

### Slash commands (`.claude/commands/`)

- `/build` ‚Äî `pnpm build` (turbo build, all workspaces).
- `/typecheck` ‚Äî `pnpm typecheck`; pass a workspace to narrow (e.g. `/typecheck @workspace/ui`).
- `/lint` ‚Äî `pnpm lint`.
- `/format` ‚Äî `pnpm format` then report what changed.
- `/generate` ‚Äî `pnpm generate` (re-run Orval to refresh `api-client/src/generated`).
- `/check` ‚Äî `pnpm check:all` (generate ‚Üí format ‚Üí typecheck ‚Üí lint ‚Üí build ‚Äî the full gate).
- `/add-ui` ‚Äî scaffold a new cross-platform primitive in `@workspace/ui` (web `.tsx` + native
  `.native.tsx` + `index.ts`) wrapping the matching `web-ui`/`native-ui` component.

### Subagents (`.claude/agents/`)

- `architecture-reviewer` ‚Äî audits a diff for boundary violations: `web-ui`/`native-ui` imported
  outside `ui`, cross-package relative imports, edits to vendored/generated code, version literals
  that should be in the catalog. Use before finishing any multi-file change.
- `ui-sync-reviewer` ‚Äî checks that every `@workspace/ui` primitive has a synchronized web + native
  variant with matching exports/props/behavior, and is registered in `index.ts`.

### Skills (`.claude/skills/`) ‚Äî invoke with `/name`, or Claude auto-invokes by description

- `/add-ui-component <Name>` ‚Äî adds a synchronized cross-platform component to `@workspace/ui`
  wrapping the `web-ui` (shadcn) and `native-ui` (rnr) implementations, with a parity checklist.

### Rules (`.claude/rules/`) ‚Äî auto-loaded by file path, no invocation needed

Path-gated guidance that loads when you open a matching file:

- `vendored-ui-components.md` ‚Üí `packages/web-ui/**`, `packages/native-ui/**`
- `ui-abstraction.md` ‚Üí `packages/ui/src/components/**`
- `generated-api.md` ‚Üí `packages/api-client/src/generated/**`
- `cross-platform.md` ‚Üí `**/*.native.{ts,tsx}`
- `typescript-style.md` ‚Üí `**/*.{ts,tsx}`
  These are guidance (like CLAUDE.md), not enforcement ‚Äî the hook + permissions in `settings.json` are
  the enforced layer.

### Hooks (configured in `.claude/settings.json`)

- **PreToolUse guard** (`hooks/guard-protected-paths.ps1`) denies edits under
  `**/node_modules/**`, `**/.next/**`, `**/.turbo/**`, `**/dist/**`, `**/build/**`, and
  `packages/api-client/src/generated/**` (Orval-owned); and **asks before editing** vendored
  component files under `packages/web-ui/src/components/**` and `packages/native-ui/src/components/**`
  (regenerated by `pnpm shadcn-update` / `pnpm rnr-update`). See `AGENTS.md` ¬ß4‚Äì¬ß5.

### Auto-loaded per-package rules

Each package/app has an `AGENTS.md` (canonical rules) and a `CLAUDE.md` stub that imports it. When
you read/edit a file in, say, `packages/ui/`, Claude Code pulls in `packages/ui/CLAUDE.md` ‚Üí
`packages/ui/AGENTS.md`. **Edit the `AGENTS.md`, not the stub.**

### Conventions reminder

- Windows + PowerShell: use `$env:VAR`, `$null`, backtick line-continuation; run scripts via `pnpm`.
- Versions live in the `catalog:` (`pnpm-workspace.yaml`), not in individual `package.json` files.
- Don't `git commit`/`push` unless asked; never bypass the husky hooks with `--no-verify`. Git hooks
  (husky: pre-commit lint-staged, pre-push full gate) and CI are described in `AGENTS.md` ¬ß11 ‚Äî these
  are separate from the Claude `PreToolUse` guard above.
